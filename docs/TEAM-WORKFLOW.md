# Hướng dẫn Vận hành Team SubAgents

Bạn là **Orchestrator / Scrum Master**. Chỉ cần 3 commands để điều phối team.

---

## 3 Commands

| #   | Command                   | Khi nào                 | Output                 |
| :-- | :------------------------ | :---------------------- | :--------------------- |
| 1   | `/team:kickoff <ý tưởng>` | Bắt đầu dự án           | Vision + Backlog       |
| 2   | `/team:analyze`           | Sau khi approve Backlog | Architecture + Plan    |
| 3   | `/team:implement <task>`  | Sau khi approve Plan    | Code + Tests + Handoff |

---

## Quy trình

### Phase 1: Kickoff

```
/team:kickoff Todo App: CRUD tasks, gán người, deadline, filter trạng thái
```

→ Đọc output → Approve → Sang Phase 2

### Phase 2: Analyze

```
/team:analyze Ưu tiên React + Node.js, database PostgreSQL
```

→ Đọc output → Approve → Sang Phase 3

### Phase 3: Implement (lặp)

```
/team:implement Sprint 1: Implement CRUD cơ bản — Create + Read tasks
```

→ Đọc `docs/HANDOFF.md` → Tạo Issues cho việc chưa xong → Gọi implement tiếp:

```
/team:implement Sprint 2: Fix ISSUE-001, ISSUE-002 + implement Update + Delete tasks
```

---

## Xử lý Handoff

Sau mỗi lần gọi `/team:implement`, agent sẽ viết `docs/HANDOFF.md`. Bạn cần:

1. **Đọc** `docs/HANDOFF.md`
2. **Tạo Issues** từ mục "Đang dở" + "Issues cần tạo"
3. **Cập nhật** `docs/PRODUCT_BACKLOG.md` với Issues mới
4. **Gọi** `/team:implement` tiếp với tasks cụ thể
