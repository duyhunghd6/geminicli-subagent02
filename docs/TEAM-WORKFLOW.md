# Hướng dẫn Vận hành Team SubAgents

3 commands. Mỗi command spawn đúng agents + git commit khi xong.

---

## 3 Commands

| #   | Command                   | Spawn                     | Git Commit             |
| :-- | :------------------------ | :------------------------ | :--------------------- |
| 1   | `/team:kickoff <ý tưởng>` | po-agent                  | `phase1(kickoff): ...` |
| 2   | `/team:analyze`           | sa-agent → qa-agent       | `phase2(analyze): ...` |
| 3   | `/team:implement <tasks>` | 2× dev-agent ∥ → qa-agent | `phase3(sprint): ...`  |

---

## Ví dụ

### Phase 1: Kickoff

```
/team:kickoff Todo App: CRUD tasks, gán người, deadline, filter
```

→ po-agent chạy → Vision + Backlog → **git commit** → Approve

### Phase 2: Analyze

```
/team:analyze Ưu tiên React + Node.js + PostgreSQL
```

→ sa-agent thiết kế → qa-agent review → **git commit** → Approve

### Phase 3: Implement (lặp)

```
/team:implement Sprint 1: Setup project + CRUD Create + CRUD Read
```

→ Orchestrator planning (4-6 tasks, chia 2 nhóm) → 2 dev-agents song song → qa-agent test → **git commit**

```
/team:implement Sprint 2: Fix ISSUE-001 + CRUD Update + Delete
```

---

## Sprint Capacity

- Mỗi dev-agent: **15 turns** → handle **2-3 tasks**
- Song song **2 dev-agents** → **4-6 tasks/sprint**
- Orchestrator chọn tasks không phụ thuộc nhau cho 2 nhóm

## Definition of Done

Dev Agent chỉ DONE khi: ✅ Unit tests viết xong → ✅ Build pass → ✅ Tests pass → ✅ Handoff viết

## Xử lý Handoff

1. Đọc `docs/HANDOFF_DEV1.md`, `HANDOFF_DEV2.md`, `HANDOFF_QA.md`
2. Việc xong → cập nhật Backlog (Done)
3. Việc dở → tạo Issues → Backlog → `/team:implement` Sprint sau
